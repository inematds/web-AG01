O que é

Um protótipo, não o portal oficial

web-AG01 é um experimento interno: uma landing page para a plataforma educacional INEMA.Club (foco em IA e Robótica), construída em React/Vite. Não é o inema.club nem o pro.inema.club — é um site separado, testando design e fluxo de conversão, hoje publicado em modo demo no Vercel.

🎨 Landing completa

Hero, metodologia, depoimentos, planos e rodapé prontos — a apresentação da proposta do INEMA.Club (aprendizado acelerado, jogos educativos EAI, comunidade INEMA.VIP, mentorias e certificados).

🤖 Diagnóstico de IA (mockado)

Ferramenta de "auditoria" que simula uma análise de potencial com IA. Em modo demo devolve dados de exemplo; com uma chave do Gemini, chama a API de verdade.

💳 Planos e preços

Seção de pricing ativa com três níveis — Básico (grátis), Premium (R$97) e VIP (R$297) — e um login/dashboard que existe no código mas fica desativado até configurar Supabase.

Como funciona

Do clone ao deploy

O projeto é um SPA Vite/React estático. Sem servidor próprio: builda pra arquivos estáticos e sobe no Vercel. As integrações externas (IA real, login) são opcionais e desligadas por padrão.

npm install→ npm run dev / build→ bundle estático (Vite)→ deploy Vercel→ (opcional) liga Gemini + Supabase
Pré-requisitos

Pra rodar localmente

Modo demo não exige nenhuma conta ou chave — só Node instalado.

Node.js 16+

Único requisito real pra rodar o site em modo demo.

# confira a versão
node -v

(Opcional) Google AI Studio

Só se quiser diagnóstico de IA de verdade em vez do mock.

# chave
VITE_GEMINI_API_KEY=...

(Opcional) Supabase

Só se quiser ativar login e dashboard, hoje desativados.

# chaves
VITE_SUPABASE_URL=...
VITE_SUPABASE_ANON_KEY=...
Guia de uso · passo a passo

Rodar o site em modo demo

Funciona 100% sem configurar nada. Bundle otimizado (~373KB).

1

Clone o repositório

Pega o código do projeto.

git clone https://github.com/inematds/web-AG01.git  # baixa o repo
cd web-AG01
2

Instale as dependências

React, Vite, Tailwind, Framer Motion e afins.

npm install  # instala node_modules
3

Suba o servidor de desenvolvimento

Abre em modo demo, com hot reload.

npm run dev  # http://localhost:3000
4

Gere o build de produção

Empacota tudo em arquivos estáticos.

npm run build  # gera dist/
npm run preview  # testa o build localmente
5

Deploy no Vercel

Importe o repositório inematds/web-AG01 em vercel.com — o deploy funciona sem nenhuma variável de ambiente, já que o site parte em modo demo.

# opcional: variáveis extras no dashboard do Vercel
VITE_GEMINI_API_KEY=...
VITE_SUPABASE_URL=...
VITE_SUPABASE_ANON_KEY=...

Direto do README do projeto — nada aqui foi inventado.

Atenção — segurança. O próprio repo documenta (SECURITY_DEPLOYMENT_GUIDE.md) que a autenticação atual guarda senha em texto plano, roda no cliente e não tem Row Level Security no Supabase. Ou seja: mesmo ativando login/dashboard, isso ainda não é produção-ready — é só o protótipo indo um passo além do modo demo.
